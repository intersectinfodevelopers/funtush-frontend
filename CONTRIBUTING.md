# Contributing to Funtush Frontend

Thank you for contributing to Funtush Frontend. Please follow these guidelines:

## Getting Started
- Open an issue before starting large changes.
- Branch from `main` and use descriptive branch names.

## Pull Requests
- Keep PRs focused on a single change.
- Include a clear summary of what changed and why.
- Add reviewers and wait for at least one approval before merging.

## Code Style
- Follow existing code patterns in the repository.
- Run linting and tests locally where applicable.

## Reporting Security Issues
- Do not disclose vulnerabilities in public issues.
- Use the security contact in `SECURITY.md`.
